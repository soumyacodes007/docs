# Operations

This page covers the operational side of PIXA: what to watch, what to verify, and what to keep small.

## Daily checks

- the wallet is funded
- the spending caps are still correct
- the hub is healthy
- hosted flows still open
- the docs still match the tool names

## Demo checks

Before a demo, verify:

1. `check_balance` works
2. `pay` works on the intended network
3. `x402_fetch` returns a useful result
4. `open_onramp` opens the browser flow
5. `rekey_with_pera` reaches the wallet approval path

## Why this is in tool reference

Operators need a short checklist more than they need theory.
This page is the practical runbook for the PIXA flow.
